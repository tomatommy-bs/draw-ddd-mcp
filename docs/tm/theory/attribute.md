# 属性の分類

根拠: 公理 A2

## 概要

T字型構造の右列に配置される属性は、以下のように分類される。

```
属性
├── 個体指定子（左列）       → identifier.md で定義
│   ├── own（固有）
│   └── reference（参照）
│
└── 状態記述属性（右列）
    ├── 記述子（通常属性）
    └── 導出属性
```

## 記述子（状態記述子）

エンティティの状態を直接記述する属性。データベースに永続化される。

- 例: 顧客名称、与信限度額、受注日、受注数
- 値はユーザーの入力またはシステムの操作により設定される

### 時間属性

Event に付随する日時・日付。記述子の一種だが、Event にとって特別な意味を持つ。

- 例: 受注日、出荷日時、支払日
- Event が「いつ起きたか」を示す
- TM 手法では Event に時間属性があることを推奨する

### 区分属性（discriminator）

サブセット分解の基準となる属性。記述子の一種。

- 例: 顧客区分コード（法人/個人を区分）
- スーパーセットの右列に配置される
- この属性の値によってどのサブセットに属するかが決まる

## 導出属性 (derived)

他の属性やエンティティから計算・導出可能な属性。永続化は任意。

- 例: 受注金額（= 品目単価 × 受注数）
- 例: 年齢（= 現在日付 − 生年月日）
- 例: 受注合計（= 受注明細.金額 の合計）

### 導出属性の特徴

| 性質 | 説明 |
|------|------|
| 計算可能 | 他のデータから再計算できる |
| 冗長 | 理論上は保存不要だが、性能のため保存することがある |
| 依存元 | 計算に使う属性（同一エンティティ内または別エンティティ） |

### データモデルでの表現

導出属性は `isDerived: true` フラグで識別する。導出元の情報はコメント等で記録する。

```
受注金額 (derived)
  導出元: 品目.品目単価 × 受注.受注数
  保存: 性能上の理由で永続化
```

## 属性の配置ルール

1. 個体指定子は必ず左列
2. 記述子・導出属性は必ず右列
3. 左列と右列をまたぐ属性は存在しない
4. 1つの属性は1つのエンティティにのみ属する（共有しない）
