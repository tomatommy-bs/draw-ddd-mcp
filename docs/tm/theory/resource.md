# Resource (R)

根拠: 公理 A1, A4

## 定義

管理対象となる「もの」。時間に依存せず存在する。「いつ起きたか」と問えない対象。

**他のエンティティを参照しない。** R の左列は own（固有指定子）のみで構成される。R 同士の関係は Correspondence (C) で表現する。

## 判定基準

- 「それは管理台帳に載るか？」→ Yes なら Resource の候補
- 「時間とともにレコードが増えるか？」→ No なら Resource（Yes なら Event の可能性）
- 「それを削除したら、関連する Event が意味を失うか？」→ Yes なら Resource

## subtype 別の構造

### basic

最も基本的な Resource。固有指定子を厳密に1つ持つ。他を参照しない。

```
顧客 (R, basic)
┌──────────────┬────────────────┐
│  顧客         │       R        │
├──────────┬───┴────────────────┤
│顧客NO    │ 顧客名称            │
│          │ 顧客区分コード       │
│          │ 与信限度額           │
└──────────┴────────────────────┘
  左列: own × 1
  reference: なし
```

### subset

親 Resource の特化。親の固有指定子を reference として継承し、own 指定子を持たない。

```
法人顧客 (R, subset)
┌──────────────────┬────────────┐
│  法人顧客          │     R      │
├──────────┬───────┴────────────┤
│顧客NO(R) │ 法人番号            │
│          │ 代表者名             │
└──────────┴────────────────────┘
  左列: reference × 1（親の own を継承）
  親: 顧客 (R, basic)
```

注: R の自己参照（部品構成 等）は Recursive (Re) 型で表現する。R の subtype ではない。
