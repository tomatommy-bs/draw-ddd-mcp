# 個体指定子 (Identifier)

根拠: 公理 A2, A3

## 定義

エンティティの個体（レコード）を一意に特定するための属性。T字型構造の左列を構成する。

## 種別

### own（固有指定子）

そのエンティティ自身が独自に採番する識別子。

- エンティティが「自分で管理するコード/番号」
- 例: 顧客NO、受注NO、品目コード

### reference（参照指定子）

他エンティティの固有指定子を借用したもの。T字型 ERD では `(R)` を付記する。

- 参照先エンティティとの関係（Reference）から自動的に導出される
- 手動で作成するものではない（公理 A6: 構造決定性）
- 例: 顧客NO(R)、品目コード(R)

## 複合識別子

2つ以上の個体指定子の組み合わせでレコードを一意に特定するパターン。

```
受注明細の識別子 = 受注NO(R) + 明細NO(own)
  → 受注NO だけでは一意にならない（同一受注に複数明細がある）
  → 明細NO だけでは一意にならない（別の受注にも明細NO=1 がある）
  → 組み合わせで初めて一意になる
```

identifierOrder で複合識別子内の順序を管理する。

## 型・subtype 別の識別子パターン

| 型 / subtype | own | reference | 構造 |
|-------------|-----|-----------|------|
| R basic | 1 | 0 | 固有指定子のみ。他を参照しない |
| R subset | 0 | 1 | 親の own を reference で継承 |
| E basic | 1+ | 1+ | own + R/E への参照 |
| E subset | 0 | 1 | 親 E の own を reference で継承 |
| C | 0-1 | 2+（R 群から） | 2つの R（or C+R）の own を借用 |
| Re | 0 | 2（同名） | 同一 R の own を2回借用 |
| M | 0 | 2（E 群から） | 2つの E の own を借用 |

## 識別子の不変性

個体指定子は原則として変更されない。変更が必要な場合、それは設計の見直しが必要なシグナルである。

## データ型

個体指定子のデータ型に制約はないが、実務上は以下が一般的:

- 連番: integer / bigint
- コード: string (固定長)
- UUID: uuid
