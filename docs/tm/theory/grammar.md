# 関係文法

エンティティ同士がどう関係するかの規則。「どう繋がるか」を規定する。

## 関係の全体像

```
┌──────────┬──────────────────────────────────────┐
│  関係     │  表現方法                              │
├──────────┼──────────────────────────────────────┤
│  R - R   │  Correspondence (C) を定義             │
│  (異なる) │  C の左列に両 R の own が (R) で出現    │
├──────────┼──────────────────────────────────────┤
│  R - R   │  Recursive (Re) を定義                 │
│  (同一)  │  Re の左列に同一 R の own が (R) × 2    │
├──────────┼──────────────────────────────────────┤
│  C - R   │  新たな C を定義（連鎖）                │
├──────────┼──────────────────────────────────────┤
│  E → R   │  E の左列に R の own が (R) で出現      │
├──────────┼──────────────────────────────────────┤
│  E → E   │  1:N → E2 が E1 を直接参照             │
│  (1:N)   │  E2 の左列に E1 の own が (R) で出現    │
├──────────┼──────────────────────────────────────┤
│  E - E   │  Mapping (M) を定義                    │
│  (N:1等) │  M の左列に両 E の own が (R) で出現     │
├──────────┼──────────────────────────────────────┤
│  R → E   │  存在しない                             │
│  R × E   │  存在しない                             │
│  E 再帰   │  存在しない                             │
└──────────┴──────────────────────────────────────┘
```

## 公理 A3: 個体指定子の種別

エンティティ間の関係は、個体指定子の種別として表現される:

| 種別 | 記法 | 意味 |
|------|------|------|
| **own（固有指定子）** | `顧客NO` | そのエンティティ自身を一意に識別する。関係を持たない |
| **reference（参照指定子）** | `顧客NO(R)` | 他エンティティの固有指定子を借用。関係の証 |

- own は「自分で採番する」—— 他のエンティティに依存しない
- reference は「相手の own をコピーする」—— 関係があるから存在する
- R の左列は own のみ（R は他を参照しない）
- E, C, M の左列は reference を含みうる

詳細: [identifier.md](./identifier.md)

## 公理 A4: R-R の関係

R 同士は直接参照しない。R-R の関係は **Correspondence (C)** または **Recursive (Re)** で表現される。

### 異なる R × R → Correspondence (C)

```
取引先品目 (C) = 取引先(R) × 品目(R)
┌──────────────┬───┐
│取引先品目      │ C │
├──────────┬──┴───┤
│取引先コード(R)│ 取引価格│
│品目コード(R)│       │
└──────────┴──────┘
  左列: 2つの R の own を reference で借用
  右列: 対照関係に付随する属性
```

### C × R → 新たな C（連鎖）

既存の Correspondence と Resource から新たな Correspondence を定義できる。

```
取引先品目 (C)  = 取引先(R) × 品目(R)
取引先品目倉庫 (C) = 取引先品目(C) × 倉庫(R)
┌──────────────┬───┐
│取引先品目倉庫   │ C │
├──────────┬──┴───┤
│取引先コード(R)│ 在庫数 │  ← 取引先品目(C)経由
│品目コード(R)│       │  ← 取引先品目(C)経由
│倉庫コード(R)│       │  ← 倉庫(R)から
└──────────┴──────┘
```

### 同一 R × R → Recursive (Re)

同一 Resource の自己参照。階層構造やネットワーク構造を表現する。

```
部品構成 (Re) = 部品(R) × 部品(R)
┌──────────────────┬────┐
│  部品構成          │ Re │
├──────────┬───────┴────┤
│部品コード(R)│ 構成数      │
│部品コード(R)│            │
└──────────┴────────────┘
  左列: 同一 R の own を2回借用（同名、identifierOrder で区別）
```

## 公理 A5: E → R の関係

Event は Resource を参照する。方向は常に E → R。R → E は存在しない。

```
受注(E) → 顧客(R)
  → 受注の左列に「顧客NO(R)」が出現

受注(E) → 品目(R)
  → 受注の左列に「品目コード(R)」が出現
```

Event は複数の R を参照できる。

## 公理 A6: E-E の関係

E-E の関係は cardinality で表現方法が決まる。

### 1:N の場合 — 直接参照

E2 が E1 の参照を持つ。

```
受注(E1) : 出荷(E2) = 1:N
  → 出荷の左列に「受注NO(R)」が出現
  → 出荷は受注を直接参照する

┌──────┬───┐     ┌────────┬───┐
│受注   │ E │     │出荷     │ E │
├──┬───┴───┤     ├────┬───┴───┤
│受注NO│受注日│────▶│受注NO(R)│出荷日│
│顧客(R)│    │ 1:N │出荷NO  │     │
└──┴──────┘     └────┴──────┘
```

### N:1 / N:M の場合 — Mapping (M) を定義

2つの E の間に Mapping を定義する。

```
受注(E1) : 請求(E2) = N:1（複数受注を1つの請求にまとめる）
  → Mapping「受注請求対応(M)」を定義

┌──────┬───┐  ┌──────────┬───┐  ┌──────┬───┐
│受注   │ E │  │受注請求対応│ M │  │請求   │ E │
├──┬───┴───┤  ├──────┬───┴───┤  ├──┬───┴───┤
│受注NO│受注日│  │受注NO(R)│     │  │請求NO│請求日│
│    │     │  │請求NO(R)│     │  │    │請求額│
└──┴──────┘  └──────┴──────┘  └──┴──────┘
               M の左列: 両 E の own を reference で借用
```

## 公理 A7: 構造決定性

参照関係（Reference）が確定すると、参照元の左列に出現する reference 指定子は**自動的に決まる**。手動で定義するものではない。

これは R, E, C, Re, M すべてに適用される:

| 場面 | 構造決定性の発現 |
|------|-----------------|
| E → R | R の own が E の左列に (R) で出現 |
| E → E (1:N) | E1 の own が E2 の左列に (R) で出現 |
| C = R × R | 両 R の own が C の左列に (R) で出現 |
| C = C × R | C の参照指定子群 + R の own が新 C の左列に出現 |
| Re = R × R (同一) | 同一 R の own が Re の左列に (R) × 2 で出現 |
| M = E × E | 両 E の own が M の左列に (R) で出現 |
| subset | 親の own が子の左列に (R) で出現 |

逆に、Reference を削除すると対応する reference 指定子も消える。

詳細: [reference.md](./reference.md)

## 公理 A8: サブセット/スーパーセット

Resource（または Event）は区分属性によりサブセットに分解できる。

```
顧客（スーパーセット）
  │ 区分属性: 顧客区分コード
  ├── 法人顧客（サブセット）
  └── 個人顧客（サブセット）
```

- スーパーセットの右列に**区分属性（discriminator）**が存在する
- 区分属性の値の数だけサブセットが分岐しうる
- サブセットは親の固有指定子を reference として継承する
- サブセットは own 指定子を持たない
- サブセットは親にはない固有の属性を右列に持つ

詳細: [subtype.md](./subtype.md)
